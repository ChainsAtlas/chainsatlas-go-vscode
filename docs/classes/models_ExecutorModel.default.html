Class default

The ExecutorModel class provides a comprehensive model to manage and execute bytecode contracts. It encapsulates the logic to handle bytecode compilation, gas estimation, and contract execution. This model primarily interacts with Ethereum-based operations using Web3.js and emits events to signal different stages or statuses of these operations.

Emits

Emitted when waiting for a bytecode structure.

Emits

Emitted to synchronize or update any listeners of state changes.

Emits

Emitted when waiting for gas estimation.

Emits

Emitted when a bytecode structure is received.

Emits

Emitted when gas amount is received.

Example

const executor = new ExecutorModel();
executor.on(ExecutorModelEvent.SYNC, () => {
    // state synchronization logic
});

Private _composeInput

  • _composeInput(bytecodeStructure, inputArgs): string

  • Composes the final bytecode by replacing placeholders with the provided input arguments.

    This method takes the base bytecode structure and substitutes the placeholders with the actual values provided in inputArgs. The placeholders are determined using the key from the bytecodeStructure and the index of the argument in inputArgs.

    Parameters

    • bytecodeStructure: BytecodeStructure

      The structure of the bytecode that contains placeholders for arguments.

    • inputArgs: number[]

      An array of arguments to replace the placeholders in the bytecode.

    Returns string

    The final composed bytecode string, prefixed with "0x".

    Throws

    • Throws an error if the number of arguments provided in inputArgs does not match the expected number of arguments (nargs) in bytecodeStructure.

  • Compiles the bytecode based on the provided number of arguments.

    This method initiates the compilation process for the bytecode associated with the user file. It sets the internal state to indicate the start of the compilation process, emits relevant events, and finally retrieves and sets the bytecode structure once compilation is complete.

runBytecode

  • runBytecode(args, from, vUnitAddress, web3): Promise<void>

  • Runs the bytecode by invoking the appropriate contract method on the blockchain.

    This method initiates the process to execute the bytecode on the blockchain. After validating the bytecode structure, it composes the input for the bytecode, estimates the gas required for the transaction, and sends the transaction. It also handles various transaction events like sending, confirmation, and errors.

    Parameters

    • args: number[]

      An array of arguments required by the bytecode.

    • from: string

      The Ethereum address from which the transaction is sent.

    • vUnitAddress: string

      The Ethereum address of the virtual unit contract.

    • web3: Web3

      An instance of the Web3 library to interact with the Ethereum blockchain.

    Returns Promise<void>

    A promise that resolves once the bytecode has been executed.

    Throws

    • Throws an error if the bytecode structure is invalid or not set.
    • Throws an error if there is any issue with the transaction or the returned data is invalid.

    Emits

    ExecutorModelEvent.SYNC - Emitted to indicate synchronization with the current state.

    Emits

    ExecutorModelEvent.WAITING_GAS - Emitted when the system is waiting for the gas estimate.

    Example

    executorModel.runBytecode(args, fromAddress, contractAddress, web3Instance);
